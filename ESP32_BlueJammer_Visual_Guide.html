What is ESP32-BlueJammer?

Advanced 2.4GHz Spectrum Disruption System

The ESP32-BlueJammer is a compact, portable wireless disruption device designed for security testing and controlled interference operations. Operating within the 2.4GHz ISM band, it generates targeted interference signals capable of disrupting a wide range of wireless communications.

üí° OLED Compatibility

Your 1.3" OLED may work with "0.96" OLED" firmware if it uses SSD1306 driver. Try it first, or use "Generic" firmware if display doesn't work.

üîß Display Shows Garbage/Corrupted?

This means display is working but firmware/display type mismatch. Try:

  1. Try different "0.96" OLED" firmware variant (Combo-Channel-Select, Basic, etc.)
  2. Check if display is SSD1306 or SH1106 (most 1.3" are SSD1306)
  3. Verify I2C address is 0x3C (most common)
  4. If nothing works, use "Generic" firmware (modules work, no display)
‚ö†Ô∏è Display Not Working?

Most Common Issues:

  1. Wrong Firmware: Make sure you flashed "0.96" OLED" firmware, NOT "Generic"
  2. Wrong Pin Connections: Verify SDA ‚Üí GPIO 4, SCL ‚Üí GPIO 5 (not swapped!)
  3. Power Issue: OLED must use 3.3V (NOT 5V!)
  4. Loose Connections: Re-seat all OLED wires

Quick Fix: Re-flash with "0.96" OLED" firmware from the web flasher

Common Issues & Solutions

üî¥ Issue #1: Missing Capacitor (MOST COMMON!)

Problem: Module 1 fails to initialize because the 10¬µF capacitor is missing or not connected.
Solution: Connect 10¬µF capacitor: + (positive) to Pin 2 (VCC), - (negative) to Pin 1 (GND) on Module 1.
Why: nRF24L01 modules need this capacitor for stable power. Without it, they cannot initialize properly.

üîå Issue 2: Loose Connections

Solution: Unplug USB, re-seat all Module 1 wires, ensure tight connections. Gently tug each wire to verify it's secure.

‚ö° Issue 3: Power Problems

Solution: Verify 3.3V is present at Module 1 VCC (use multimeter). Check capacitor is connected correctly. Try different USB port/cable for more power.

üî¢ Issue 4: Wrong Pin Mapping

Solution: Double-check each pin connection matches the table above exactly. Common mistake: swapping CE and CSN pins.

‚ö†Ô∏è Issue 5: GPIO 12 Boot Problem

Solution: GPIO 12 (MISO) can cause boot issues if pulled HIGH. Temporarily disconnect GPIO 12, boot ESP32, then reconnect. Or verify GPIO 12 connection is clean.

üîß Issue 6: Faulty Module

Solution: If Module 2 works but Module 1 doesn't, try swapping them. If Module 2 works in Module 1's position, Module 1 hardware may be faulty.

üí° Pro Tip: Compare Working vs Non-Working

Since Module 2 (VSPI) is working, compare its connections to Module 1. Both should have identical wiring patterns, just different GPIO pins. If Module 2's connections look different, that's your clue!
