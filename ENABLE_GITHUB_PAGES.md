# How to Enable GitHub Pages - Step by Step

## üìã Step-by-Step Instructions

### Step 1: Go to Your Repository
1. Open your browser
2. Go to: **https://github.com/JulietKiloCharlie/BlueJammer**
3. Make sure you're logged into GitHub

### Step 2: Open Settings
1. Look at the **top menu bar** of your repository
2. You'll see tabs: **Code**, **Issues**, **Pull requests**, **Actions**, **Projects**, **Wiki**, **Security**, **Insights**
3. Click on **"Settings"** (it's usually the rightmost tab, or near the end)

### Step 3: Find Pages Section
1. In the **left sidebar** of the Settings page, scroll down
2. Look for **"Pages"** in the list (it's usually near the bottom)
3. Click on **"Pages"**

### Step 4: Configure Pages
1. Under **"Source"** section, you'll see a dropdown
2. Click the dropdown that says **"None"** or is empty
3. Select **"Deploy from a branch"**

### Step 5: Select Branch and Folder
1. A new dropdown will appear for **"Branch"**
2. Select **"main"** from the dropdown
3. Another dropdown for **"Folder"** will appear
4. Select **"/ (root)"** from the folder dropdown
5. Click the **"Save"** button

### Step 6: Wait for Deployment
1. After clicking Save, you'll see a message that Pages is being deployed
2. Wait 1-2 minutes
3. You'll see a green checkmark when it's ready
4. Your site URL will appear at the top: `https://julietkilocharlie.github.io/BlueJammer/`

## ‚ùì Troubleshooting

**Can't find Settings?**
- Make sure you're logged into GitHub
- Make sure you're the owner of the repository
- Settings tab should be visible in the repository menu

**No "Pages" option?**
- Make sure you're in the Settings page
- Scroll down the left sidebar - Pages is usually near the bottom
- It might be under "Code and automation" section

**Still having issues?**
- Try refreshing the page
- Make sure the repository is set to **Public** (required for free GitHub Pages)
